# Filecoin Specification

**One-line:** Filecoin is a decentralized storage network and blockchain whose consensus power is derived from verifiable, useful storage rather than wasted computation, turning disk space into a market commodity and a Sybil-resistance mechanism.

## Summary

[[filecoin]] is a distributed storage network built on a blockchain in which miners earn the native currency, [[fil-token]] (⨎), by provably dedicating storage capacity to the network. Unlike Nakamoto proof-of-work, Filecoin's consensus uses [[proof-of-storage]] itself: a miner's influence over consensus is proportional to the storage it provides and continually proves. The chain doubles as a marketplace — clients pay FIL to store files via on-chain [[storage-market]] deals, and the protocol enforces those deals over time without further interaction from the original parties. Filecoin is closely tied to the broader [[ipfs]] / [[content-addressing]] ecosystem: client files are converted to UnixFS [[merkle-dag]] structures, identified by CIDs, and the protocol's own data structures are stored as [[ipld]] DAG-CBOR.

Architecturally, Filecoin is organized as a set of subsystems running on nodes, and a set of on-chain actors. A node is a full software/hardware system running components (ChainSync, message pool, VM, miner subsystems) and exposing APIs; subsystems are conceptual divisions (storage market, retrieval market, VM). Actors are the on-chain equivalent of smart contracts: each carries a FIL balance, a `state` pointer, a `code` CID identifying its type, and a `nonce`. There are eleven built-in system actors, including the Account, Init, Cron, Reward, Multisig, Storage Power, Storage Market, Storage Miner, Verified Registry, and Payment Channel actors.

The [[filecoin-vm]] is a replicated state machine that executes actor code; every on-chain operation incurs a gas cost. The global state is a [[merkle-dag]] State Tree of actors, identified by a CID stored in the IPLD store. State transitions are driven by `messages`: a message bundles a value transfer plus an optional method invocation with parameters. External users submit `SignedMessage`s (paying gas to the including miner); actors may synchronously call other actors only as a side-effect of a top-level user message. Messages undergo syntactic validation (well-formed addresses, non-negative value/gas) and semantic validation (signature verifies against the `From` account's key), producing a `MessageReceipt` with an exit code and gas used. The [[actor-model]] underlies the entire VM.

Storage is committed in fixed-size [[sector]]s (32 GiB or 64 GiB, chosen by the miner). A sector is filled either with client deal data ("regular"/"verified" sectors) or arbitrary data (Committed Capacity / CC). Before a sector earns power it must be [[sealing|sealed]] via [[proof-of-replication]] (PoRep): a slow, computation-heavy encoding that produces a unique replica tied to (i) the data, (ii) the miner's identity (ProverID), and (iii) the time of sealing (via `SealRandomness` drawn from the VRF chain to prevent long-range/replay attacks). Sealing yields the unsealed-data commitment CommD and the sealed replica commitment CommR; a [[zk-snark]] (Groth16) compresses the bulky vanilla proof for on-chain submission. Filecoin launched with [[stacked-drg]] PoRep (SDR), designed by Ben Fisch (EUROCRYPT19).

[[stacked-drg]] (SDR) enforces sequential, non-parallelizable, regeneration-resistant encoding by stacking `LAYERS` (10) depth-robust graphs of `GRAPH_SIZE` (2^31) nodes, each adjacent pair connected by a bipartite expander graph. DRG parents are generated by Bucket Sampling (base degree 6); expander parents via a Feistel-network pseudorandom permutation (expansion degree 8, 13 parents total). Each node is labeled with SHA254 over its parents' labels; the final layer's labels form a key that encodes the data into the replica. CommC commits to the column labels, CommRLast to the replica, and CommR = H(CommC ‖ CommRLast). Merkle trees use the [[poseidon-hash]] (arity 2/4/8/11 → widths t=3/5/9/12) over the BLS12-381 scalar field.

Once sealed, miners continuously prove storage via [[proof-of-spacetime]] (PoSt), in two forms. **WinningPoSt** is demanded only of miners elected to mine a block in a given epoch; it proves a live replica at a specific instant, must be returned within a short deadline, and grants the block reward (no penalty for missing it beyond losing the reward). **WindowPoSt** is the audit: each 24-hour proving period is split into 48 half-hour deadlines; a miner's sectors are partitioned (2349 sectors per partition, 10 challenges each) and one SNARK proof per partition is published per deadline so every sector is audited at least once daily. Missing WindowPoSt produces a [[fault]] and slashing.

Consensus is [[expected-consensus]] (EC), a probabilistic Byzantine-fault-tolerant leader election. Each epoch (30 s) every miner runs a [[bls-signatures|VRF]]-based election seeded by [[drand]] beacon randomness; a miner wins roughly in proportion to its quality-adjusted power via Poisson sortition, yielding a `WinCount` (allowing a single lucky block to earn multiple rewards while remaining Sybil-invariant). All winning blocks at a height form a `tipset`; the chain is the heaviest (greatest cumulative weight) chain, where weight rewards both total power and the number of blocks per round. EC enforces soft finality (≈900 epochs) and slashes consensus faults (double-fork, time-offset, parent-grinding). The [[storage-power-consensus]] (SPC) subsystem maintains the Power Table mapping each miner to its quality-adjusted power and brokers EC, randomness tickets, and chain selection. ChainSync handles bootstrap, catch-up, and follow modes over libp2p GossipSub/BlockSync/Bitswap/GraphSync with progressive (BV0–BV8) block validation.

The [[fil-token]] economy combines collateral and minting. Miners post three kinds of [[collateral]]: an initial pledge (a storage-pledge ≈20 days of expected sector block reward plus a consensus-pledge targeting ~30% of circulating supply locked, scaled by sector QAP over the network baseline), block-reward-as-collateral (rewards vest linearly to back commitments and are slashed on early termination), and storage-deal provider collateral. Block reward minting blends **simple minting** (6-year half-life exponential decay of 30% of the 1.1×10^9 FIL mining allocation) with **baseline minting** (70%, released along an exponential baseline of network raw-byte power via "effective network time"). Per-round reward is divided by the expected leaders E=5 and multiplied by each winner's WinCount. [[payment-channels]] enable off-chain retrieval micropayments (lanes, vouchers) settled on-chain.

Cryptography rests on a handful of primitives: [[poseidon-hash]] (a SNARK-friendly sponge over BLS12-381's scalar field, instances (p,M=128,t)) for all Merkle trees and CommR; SHA254 for SDR labeling; Blake2b for randomness-seed derivation, Feistel rounds, and address checksums; [[drand]] for unbiasable public randomness combined with domain-separation tags into beacon entries; [[bls-signatures]] over BLS12-381 (G1 keys, G2 signatures — deterministic and aggregatable) for blocks/tickets/messages, plus secp256k1 ECDSA for external compatibility. Addresses are protocol-tagged (0=ID, 1=secp256k1, 2=actor, 3=BLS) with network prefix and Blake2b checksum. The library stack is [[libp2p]] (Kademlia DHT, GossipSub, PeerID/Multiaddress), [[ipld]] (DAG-CBOR, CAR files, GraphSync), [[ipfs]]/UnixFS, [[multiformats]] (CIDs), and [[drand]]. Reference implementations include Lotus (Go), Venus (Go), Forest (Rust), and Fuhon (C++).

## Key takeaways
- Consensus power = provable, *useful* storage: PoRep + PoSt replace proof-of-work, making storage the Sybil-resistance resource ("useful proof of work").
- Two distinct PoSt roles: WinningPoSt (instantaneous, gates block production, reward-only) vs WindowPoSt (continuous 24h audit, faults trigger slashing).
- Sealing (SDR PoRep) is deliberately slow and non-parallelizable (stacked depth-robust graphs + expanders) so a replica cannot be cheaply regenerated on challenge; SNARKs compress proofs for the chain.
- The blockchain is a chain of *tipsets* (all blocks at a height), not blocks; chain selection is heaviest-weight (GHOST-style), with VRF-ticket tie-breaks and ~900-epoch soft finality.
- Power is *quality-adjusted*: deal type (CC / regular / verified) multiplies raw-byte power, incentivizing storage of useful and verified-client data.
- Economic security blends hardware cost with token collateral (initial pledge + vesting block rewards + deal collateral) and slashing for storage and consensus faults.
- Minting mixes simple exponential decay (30%) with baseline minting (70%) tied to actual network storage growth, smoothing emissions and rewarding real capacity.
- Filecoin is built on the IPFS/IPLD content-addressing stack: client data → UnixFS DAG → CAR → Piece (CommP) → Sector (CommD) → sealed replica (CommR); internal state is DAG-CBOR IPLD.
- Poseidon (over BLS12-381 scalar field) is the SNARK-friendly hash powering all Merkle trees; BLS signatures (aggregatable, deterministic) secure blocks and messages.

## Notable claims / data
- Sector sizes: 32 GiB or 64 GiB (miner's choice). SDR public params reference SECTOR_SIZE = 68,719,476,736 bytes (64 GiB), GRAPH_SIZE = 2,147,483,648 nodes, NODE_SIZE = 32 B, TREE_DEPTH = 31.
- SDR: LAYERS = 10, BASE_DEGREE = 6, EXPANSION_DEGREE = 8, PARENTS_COUNT = 13; labeling hash SHA254; expander PRP is a Feistel network keyed with Blake2b over 2^33 edges.
- Poseidon: prime field = BLS12-381 scalar field (p ≈ 2^255), security M = 128 bits, S-box exponent α = 5; widths t ∈ {3,5,9,12} for arities {2,4,8,11}; t=3 also derives CommR.
- Epoch / round = 30 s; finality F ≈ 900 epochs; ExpectedLeadersPerEpoch E = 5 (Poisson sortition).
- WindowPoSt: 24h proving period, 48 deadlines of 30 min each; partition = 2349 sectors, 10 challenges per sector; ~48 partitions ≈ 3.8 PiB to fill one partition per deadline.
- Consensus faults: double-fork mining, time-offset mining, parent-grinding → miner suspension + loss of all pledge collateral; slasher reward grows exponentially with delay.
- Token: FIL_BASE = 2×10^9 FIL; mining allocation M∞ = 55% = 1.1×10^9 FIL; minting mix γ = 0.7 (70% baseline / 30% simple); simple-minting half-life = 6 years (λ ≈ 3.663×10^-9 Hz); epoch τ = 30 s.
- Initial pledge = storage pledge (≈ Estimated20DaysSectorBlockReward) + consensus pledge (30% × FILCirculatingSupply × SectorQAP / max(NetworkBaseline, NetworkQAP)).
- Storage-fault penalties: Fault Fee (per offline day), Sector Penalty (undeclared fault at WindowPoSt), Termination Penalty.
- Signatures: BLS12-381 (G1 public keys, G2 signatures, aggregatable & deterministic) + secp256k1 ECDSA (external compat). Addresses: protocol 0=ID, 1=secp256k1 (Blake2b-160 of 65-byte pubkey), 2=actor, 3=BLS (48-byte pubkey); checksum = Blake2b-4; prefix f (mainnet) / t (testnet).
- Data pipeline: file → UnixFS DAG (Payload CID) → .car → padded → Piece (Piece CID / CommP) → Sector (CommD, unsealed) → sealed replica (CommR).
- 11 built-in system actors; internal state stored as IPLD DAG-CBOR; HAMT and AMT used for collections.
- Planned PoRep upgrade: SDR → Narrow Stacked Expander (NSE) for lower sealing cost and retrieval latency.
